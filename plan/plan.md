# 交易策略与回测优化计划

## 当前现状

### 策略定义现状

- 当前买入策略已拆分为两类：
  - 趋势建立初期型：关注缩量回踩、低乖离、守 `MA20`、重新放量。
  - 突破加速型：关注二次金叉、前高附近、量能放大、次日确认。
- 当前卖出策略暂时统一：
  - 固定止损 `10%`。
  - 后续任一交易日收盘价跌破 `MA20` 即离场。
  - 获利且因跌破 `MA20` 离场后，冷却 `30` 天。
- 回测层已支持通过参数控制单独回测某一种买入策略，或直接对比两种买入策略表现。

### 信号识别现状

- 信号主入口仍为 `stock/compute_indicators.py` 中的 `analyze_stock`。
- 当前输出结果中已增加：
  - `strategy_setups`
  - `strategy_labels`
  - `primary_strategy`
  - `primary_strategy_label`
- 当前策略分类逻辑：
  - 趋势建立初期型允许“首次金叉”或“二次金叉”后，在较低乖离、至少一次缩量回踩、上涨放量且守住 `MA20` 时入选。
  - 突破加速型要求“二次金叉”且至少已有两次缩量回踩，同时靠近前高并满足放量条件。

### 回测执行现状

- `stock/backtest.py` 已支持：
  - `--strategy-filter all|trend_init|breakout_accel`
  - `--compare-strategies`
- 当前入场执行逻辑：
  - 趋势建立初期型按信号当日收盘价买入。
  - 突破加速型要求次日收盘确认站上信号日高点后，按次日收盘价买入。
- 当前组合回测仍使用统一资金管理：
  - 固定单笔仓位
  - 最大持仓数限制
  - 买卖佣金与卖出印花税

### 当前主要问题

- “两次缩量回踩”的结构定义仍不完整，目前更接近“缩量下跌次数统计”，尚未完全表达“横盘中的缩量回踩，代表抛压减弱”的原始交易含义。
- 两类买入策略虽然已拆开，但分类规则仍是第一版，尚未经过分层样本验证。
- 回测已支持双策略，但结果解释层仍偏基础，尚未形成针对两类策略的专项对比报表与诊断指标。

## 策略优化

### 一、完善“两次缩量回踩”的结构定义

目标：把“次数统计”升级为“结构识别”。

建议优化方向：

- 识别“横盘区间”而不是只看单日涨跌。
- 在横盘区间内识别至少两次独立的缩量回踩，而不是累计若干缩量阴线。
- 要求回踩期间价格波动收敛，体现筹码稳定。
- 要求回踩低点不明显破坏平台结构。
- 在突破前要求出现重新放量，证明不是无量虚拉。

建议新增约束：

- 横盘窗口长度下限，例如 `5-15` 个交易日。
- 横盘区间内收盘价振幅上限。
- 两次回踩之间存在反弹间隔。
- 回踩量能低于平台均量或前一波下跌均量。
- 突破日量能显著高于横盘回踩阶段最大成交量。

### 二、趋势建立初期型继续细化

目标：提高“新趋势刚启动”样本的纯度。

建议优化方向：

- 限制首次/二次金叉距当前的时间窗口，避免太老的金叉继续算作“初期”。
- 增加“回踩不破 `MA20`，最好贴近 `MA10` 或 `MA20`”的结构约束。
- 将“至少一次缩量回踩”进一步定义为一次清晰的回撤波段，而非单根缩量阴线。
- 对重新放量设置分层阈值，区分弱修复和强修复。

建议验证维度：

- 首次金叉 vs 二次金叉
- 回踩 1 次 vs 回踩多次
- `bias < 3%` / `3%-5%` / `5%-8%`
- 放量比例 `1.0-1.5` / `1.5-2.0` / `>2.0`

### 三、突破加速型继续细化

目标：减少假突破，保留真正准备加速的样本。

建议优化方向：

- 将“两次缩量回踩”与“平台临近前高”结合起来，而不是单独看其中之一。
- 增加平台突破前的价格压缩程度判断。
- 次日确认可以扩展为：
  - 次日收盘站上信号日高点
  - 或次日站稳平台上沿
- 对突破加速型增加失败退出规则候选，用于后续比较：
  - 突破后 `2-3` 天不能继续走强则离场
  - 或重新跌回平台则离场

### 四、优先做分层回测，而不是继续调主观分值

目标：确认真正有贡献的条件组合。

建议优先输出的对比维度：

- 趋势建立初期型 vs 突破加速型
- 单独策略 vs 混合策略
- 是否存在两次缩量回踩结构
- 是否靠近前高
- 是否多头排列
- 次日确认前后收益分布差异

核心原则：

- 先找有效结构，再调权重。
- 先找收益来源，再调阈值。
- 避免用越来越多的经验打分掩盖结构定义不清的问题。

## 代码逻辑优化

### 一、信号层优化

当前状态：

- 策略分类逻辑已经进入 `compute_indicators.py`。

下一步建议：

- 把“结构识别函数”和“打分函数”继续解耦。
- 将以下逻辑拆成独立函数，便于测试和复用：
  - 横盘平台识别
  - 缩量回踩波段识别
  - 趋势建立初期型识别
  - 突破加速型识别
- 减少 `analyze_stock` 中过多条件堆叠，提升可读性。

建议目标结构：

- `detect_base_signal_features(...)`
- `detect_trend_init_setup(...)`
- `detect_breakout_accel_setup(...)`
- `score_signal(...)`
- `build_signal_payload(...)`

### 二、回测层优化

当前状态：

- `backtest.py` 已按策略过滤，并支持双策略对比。

下一步建议：

- 把“信号生成”和“交易执行”继续分层。
- 把不同买入策略的入场规则封装成独立 entry planner。
- 为未来扩展不同卖出策略预留策略级别配置，而不是写死在统一流程里。

建议目标结构：

- `build_signal_universe(...)`
- `filter_signals_by_strategy(...)`
- `build_entry_plan(...)`
- `simulate_trade_from_entry_plan(...)`
- `build_strategy_report(...)`

### 三、回测结果输出优化

建议增加以下输出：

- 按策略类型分组的交易数、胜率、平均收益、盈亏比、持仓天数。
- 按 `primary_strategy` 输出资金曲线。
- 统计“信号出现但未成交”的原因：
  - 次日未确认
  - 资金不足
  - 已持仓
  - 冷却期内
- 统计“突破加速型”中次日确认失败比例。

### 四、测试与验证优化

建议补充最小测试集：

- 给 `compute_indicators.py` 增加策略分类单元测试。
- 给 `backtest.py` 增加入场规则测试：
  - 趋势建立初期型应当当日收盘买入。
  - 突破加速型只有次日确认后才买入。
  - 次日未确认时应跳过。
- 增加少量固定 K 线样本测试，避免后续重构时把策略语义改坏。

## 建议执行顺序

### 第一阶段：先验证当前双策略拆分是否有明显差异

- 跑 `trend_init` 单独回测。
- 跑 `breakout_accel` 单独回测。
- 跑 `--compare-strategies`。
- 先判断两类策略是否具有明显不同的收益分布和持仓特征。

### 第二阶段：完善“两次缩量回踩”的结构识别

- 把“横盘 + 两次缩量回踩 + 放量突破”实现成独立检测逻辑。
- 对比新旧定义下，突破加速型样本数量与收益质量变化。

### 第三阶段：补专项统计与单元测试

- 补交易原因统计。
- 补策略维度报表。
- 补核心分类与入场测试。

### 第四阶段：再决定是否调整权重

- 只有在结构定义稳定后，再回头调整评分项和阈值。
- 若某些分值没有稳定贡献，应优先删除而不是继续堆规则。
